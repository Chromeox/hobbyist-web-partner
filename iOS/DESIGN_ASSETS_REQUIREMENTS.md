# Design Assets Requirements for Hobbyist iOS App

## Overview
Complete requirements for all visual assets needed for TestFlight deployment, App Store submission, and marketing. Organized by priority and delivery phases.

**Technical Specifications:**
- Format: PNG (no transparency for app icons)
- Color Space: sRGB or P3
- No alpha channel (solid background)
- Square dimensions exactly
- High-quality, crisp edges
- Consistent visual identity across all sizes

**Design Guidelines:**
- Simple, recognizable at small sizes
- Reflects creative/hobby theme
- Works well on both light/dark backgrounds
- Avoids text that becomes illegible at small sizes
- Professional appearance suitable for App Store

### 1.2 Launch Screen Assets
**Simple launch screen configuration:**
- Background color: Brand primary color or white
- Optional: Simple logo/wordmark (vectorized)
- No complex layouts or loading indicators
- Must work in portrait and landscape orientations

**Deliverable:**
- Launch screen background color (hex code)
- Optional simple logo SVG for launch screen

## Phase 2: App Store Marketing Assets (Required for App Store Release)

### 2.1 App Store Screenshots
**iPhone Screenshots (REQUIRED):**

**iPhone 6.7" (Pro Max) - Primary Platform:**
- Dimensions: 1290×2796 pixels
- Quantity: 3-5 screenshots
- Format: PNG or JPEG (PNG preferred)

**iPhone 6.5" (Plus models):**
- Dimensions: 1242×2688 pixels  
- Quantity: 3-5 screenshots
- Format: PNG or JPEG

**Screenshot Content Strategy:**
1. **Hero Screenshot**: Home screen with class discovery
2. **Feature Screenshot**: Class details and booking interface
3. **Profile Screenshot**: User profile and following features
4. **Search Screenshot**: Search and filtering functionality
5. **Success Screenshot**: Booking confirmation and payments

**iPad Screenshots (OPTIONAL but recommended):**
- iPad Pro 12.9": 2048×2732 pixels
- iPad 10.9": 1640×2360 pixels
- Same content as iPhone, adapted for tablet layout

### 2.2 Screenshot Design Requirements
**Visual Design Standards:**
- High-quality, realistic UI mockups
- Consistent with actual app interface
- Show diverse users and inclusive representation
- Professional photography quality
- Clean, modern aesthetic
- Vancouver-focused content where possible

**Content Requirements:**
- Real class names and studio information
- Authentic-looking user profiles
- Realistic pricing and schedules
- High-quality class/studio photos
- Professional instructor headshots
- Diverse activity categories (pottery, painting, fitness, dance)

**Text Overlay Options:**
- Feature callouts (optional)
- Benefit statements (optional)  
- Keep text minimal and readable
- Use brand typography consistently

### 2.3 App Preview Video (OPTIONAL - High Impact)
**Specifications:**
- Duration: 15-30 seconds maximum
- Resolution: Same as screenshot requirements
- Format: .mov or .mp4
- Frame Rate: 30fps
- No spoken dialogue (music/sound effects OK)

**Content Storyboard:**
1. Open app (2 seconds)
2. Browse classes (5 seconds)
3. View class details (3 seconds)
4. Complete booking (5 seconds)
5. Success/confirmation (3 seconds)
6. App logo/brand end frame (2 seconds)
